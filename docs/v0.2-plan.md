> 状态：草案 · 2026-04-17
> v0.1 收尾：`v0.1-static-all-screens` tag · feat/planner-ux 分支

## 一句话定位

v0.1 是 **8 屏静态 mock**（视觉设计系统压测 + 交互骨架验证）。
v0.2 的唯一目标是把数据真实存起来、让交互真实生效 —— 暂不做跨平台、桌面端、同步后端。

## v0.2 scope

### 核心交付物

**1. 数据层 · SQLite + Drizzle**
- Web：`wa-sqlite` + OPFS（Origin Private File System）做持久化
- `packages/db` 新包：Drizzle schema（§10 types → 表）+ migrations
- `packages/core` 扩展：事件日志（append-only）+ snapshot 机制
- 本地加密（libsodium secretbox）+ 密码短语派生（Argon2id）
- **替换所有 localStorage / useState 临时状态**

**2. Event log + 编辑会话**
- §5.3.1 Edit Session 真正实装：所有 mutation 进事件日志 + session id tagging
- `⤺ 撤销本次编辑` 变成 replay inverse events，不是 state rollback
- 15 min idle 后 session 自动归档（定时器）
- 跨视图共享的 session 注册表

**3. 路由**
- 换掉 App.tsx 的 `useState<NavKey>` 为 `react-router` 或 `@tanstack/router`
- URL 反映当前视图 + 选中态（`/cycle/c1` / `/projects/line-dayrail` / `/settings/sync`）
- 浏览器后退 / 前进生效
- Deep linking

**4. Today Track 真实 check-in 流**
- 每次 App 打开时扫描近 30 min 内结束的未标记 RailInstance
- 超过 24h 沉入 Pending 队列（§5.7）
- `Shift...` 打开真实的 Shift 标签 sheet（§5.2 C 组）+ tag 频次推荐
- 动作最终写入事件日志

**5. 打包**
- PWA manifest + Service Worker（Workbox）
- 可"Add to Home Screen"
- 离线工作

**6. Calendar 月视图**
- 把 v0.1 留下的尾巴收掉
- Calendar Rules 真正生效（星期 / 循环 / 范围 / 单日）
- 从 CycleView 主 day header 切换 template 实际写入 CalendarRule

### 不做（延后到 v0.3+）

- ❌ Sync backend 接入（Google Drive 等）—— v0.8+
- ❌ AI 实际 OpenRouter 调用 —— v0.5
- ❌ Habit Line · Phase 过渡 —— v0.4
- ❌ Desktop（Tauri）—— v0.6
- ❌ Mobile（Capacitor）—— v0.7
- ❌ 响应式移动端布局 —— v0.3 打底
- ❌ 深色模式实际切换（token 系统已备好 `sandDark` 系列，但未接通 class toggle）—— v0.2.1 可选加

### 不明但需要决策的

1. **事件日志的 Schema**：每条事件是 {aggregateId, type, payload, clock, sessionId?}？clock 用 HLC 还是 Lamport？
2. **snapshot 触发时机**：每 N 条事件、每 N 分钟、手动？
3. **undo 粒度**：session 级（§5.3.1 已定）+ 单步？还是仅 session 级？
4. **Shift sheet 的 reason 字段**：textarea 600 字？Markdown？纯文本？
