## v0.2 scope

### 核心交付物

**1. 数据层 · SQLite + Drizzle**
- Web：`wa-sqlite` + OPFS（Origin Private File System）做持久化
- `packages/db` 新包：Drizzle schema（§10 types → 表）+ migrations
- `packages/core` 扩展：事件日志（append-only）+ snapshot 机制
- 本地加密（libsodium secretbox）+ 密码短语派生（Argon2id）
- **替换所有 localStorage / useState 临时状态**

**2. Event log + 编辑会话**
- §5.3.1 Edit Session 真正实装：所有 mutation 进事件日志 + session id tagging
- `⤺ 撤销本次编辑` 变成 replay inverse events，不是 state rollback
- 15 min idle 后 session 自动归档（定时器）
- 跨视图共享的 session 注册表

**3. 路由**
- 换掉 App.tsx 的 `useState<NavKey>` 为 **`react-router-dom` v6**（2026-04-18 决定）—— tanstack-router 的类型化 params 卖点对当前复杂度溢价过高，react-router-dom 生态大 / 学习曲线低；等到 URL 复杂化到需要 typed deep-link 再换。
- URL 结构（v0.2 scope）：
  - `/` → Today Track
  - `/cycle` → Cycle View（默认本周）
  - `/tasks` / `/tasks/inbox` / `/tasks/line/:lineId` / `/tasks/archived` / `/tasks/trash`
  - `/review` / `/pending` / `/calendar`（Calendar 留占位，v0.2.1 真实装）
  - `/templates` / `/templates/:key`
  - `/settings` / `/settings/:section`（`section` ∈ appearance / sync / ai / advanced / about）
- **进 URL 的选中态**：Tasks selection、Settings section、Template tab —— 这些 deep-link 价值高。
- **不进 URL**：搜索框、过滤 chip、Cycle View 的 anchorDate —— 复杂度 vs 价值不划算；v0.2 留本地 state，后续有需要再拿出来。
- 浏览器后退 / 前进生效；刷新任何路径都停留在对应页面（SPA fallback 配 vite dev + prod 都要处理）。

**4. Today Track 真实 check-in 流**

- **三按钮动作**：`完成 / 以后再说 / 归档`（2026-04-17 决策，旧的 Shift sheet 下架）
- 点击后 6 秒 Reason toast：3 枚快速 tag chip + Undo（无强制 reason 字段）

### 已拍板（2026-04-17 讨论）

**1. 事件日志 schema · HLC**

```ts
interface HLC { wall: number; logical: number }  // wall = ms since epoch

interface Event {
  id: string               // ULID-like
  aggregateId: string      // 归属实体（Rail / Cycle / Slot ...）
  type: string             // 'rail.updated' / 'slot.assigned' ...
  payload: Record<string, unknown>
  hlc: HLC
  sessionId?: string       // 属于某个编辑会话（§5.3.1）时填
}

// HLC next
function next(prev: HLC, now = Date.now()): HLC {
  return now > prev.wall
    ? { wall: now, logical: 0 }
    : { wall: prev.wall, logical: prev.logical + 1 };
}
```

理由：DayRail 的 UI 查询强依赖墙钟（Pending `>7 天` / Review `本周 done 率`），Lamport 会丢 wall-clock 语义。HLC 比 Lamport 多 ~20 行代码，一次性成本，之后所有跨设备 replay 都稳。

**2. Snapshot 策略 · 混合：on-close + 500 事件兜底**

```ts
// 正常用户：每 session 关闭时快照
addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'hidden') {
    if (unsnapshottedCount() > 0) writeSnapshot();
  }
});

// 重度用户兜底
if (unsnapshottedCount() > 500) writeSnapshot();
```

每天事件量预估 20-100 条；正常用户每 session 1 次快照，冷启动 replay 只走当次事件 = <200ms。重度用户 500 上限兜底。

**3. Undo 粒度 · 仅 session 级，`⤺ 撤销本次编辑` 独占入口**

- **不绑 Cmd+Z** —— session 级 undo 会擦掉 N 处改动，Cmd+Z 误触风险太高
- Cmd+Z 单步 undo 违反 §5.3.1 atomic-batch 语义，且带两套 undo infra
- 用户必须显式点按钮 —— 学习曲线稍陡但零误触
- v0.3+ 如果用户真反馈再议（会变成 §11 "仍开放的问题"）

**4. Shift reason · v0.2 不采集自由文本**

2026-04-17 更新：原 500 字纯文本 textarea 随 Shift sheet 整体下架。理由：Reason toast 的 3 枚快速 tag chip 足以覆盖高频场景（晨跑没跑、会议冲突等）；真需要写大段备注的用户，留到 v0.3 Pending 详情页再补一个。

**5. Check-in 三按钮 · 替换旧的 完成/跳过/Shift/忽略**

- **完成** → `status → done`
- **以后再说** → `status → deferred`（入 Pending 队列）
- **归档** → `status → archived`（终态，循环 Rail 的次日模板不受影响）

原 `忽略` 动作被认为语义模糊（"只是本周期不要露出"还是"永久丢掉"？）直接砍掉。原 Shift sheet 的四子动作里：`Postpone` 交给 Cycle View 拖拽；`Replace / Swap / Resize / Add note` 留到 v0.3 重评。

**6. RailInstance.status 精简**

`'pending' | 'active' | 'done' | 'skipped'` → `'pending' | 'done' | 'deferred' | 'archived'`

- `active` 去掉：纯墙钟派生（`plannedStart ≤ now ≤ plannedEnd && status='pending'`）
- `skipped` 去掉：语义被 `archived`（终态放弃）和 `deferred`（半终态延后）分别吸收

**7. Signal.response 同步精简**

`'done' | 'skip' | 'shift' | 'ignore'` → `'done' | 'defer' | 'archive'`。

**8. Shift.type 同步精简**

`'postpone' | 'swap' | 'skip' | 'resize' | 'replace'` → `'defer' | 'archive'`。

**9. 旧的 reason 字段（原第 4 点）· 纯文本 500 字 + 实时字数**

- 不支持 Markdown（场景是"去接朋友 / 头疼没精神"短句，无格式需求）
- 500 字上限 = tweet 长度，"一句能说清的事"
- URL 在渲染时 regex 自动识别成可点链接
- 剩余 < 50 字时，字数计数从 tertiary 变 warn amber

对应 §10 `type Shift` 的 `reason` 字段要加 `/** max 500 chars */` 注释。
